## Оплата СБП (только из корзины)

### 1. База данных (миграция)
- Новая таблица `sbp_requisites` (одна запись, key='current'): `bank`, `card`, `holder_name`, `phone`, `updated_at`.
- Новая таблица `sbp_payments`: `id`, `order_id`, `telegram_id`, `amount_usd`, `amount_rub`, `rate` (=80), `receipt_url`, `status` ('awaiting_receipt' / 'pending_review' / 'approved' / 'rejected'), `reject_reason`, `created_at`, `reviewed_at`, `reviewed_by`.
- Storage bucket `sbp-receipts` (приватный), RLS — пользователь видит только свои файлы; service role — всё.
- Курс жёстко 80 ₽/$1 (без CryptoBot API).

### 2. Логотип СБП
- Добавить `src/assets/sbp-logo.png` (классический СБП-логотип).

### 3. Frontend — Checkout (только из корзины, не для direct/Stars/Premium)
В `Checkout.tsx`:
- Если `isDirect` (Stars/Premium) — не показывать СБП, оставить только CryptoBot/баланс (как сейчас).
- Иначе добавить переключатель способов оплаты: **CryptoBot** | **СБП** (две карточки в ряд, как на скриншоте).
- При выборе СБП кнопка «Оплатить» → переход на новую страницу `/sbp-payment?order=...`.

### 4. Frontend — новая страница `/sbp-payment`
Соответствует скриншотам:
- Шапка с балансом, переключатель CryptoBot/СБП (СБП активна).
- Карточка «Реквизиты для перевода»: сумма в ₽ (= USD × 80), курс, банк, номер карты, ФИО, телефон — каждое поле с кнопкой копирования.
- Кнопка «Я оплатил — XXX ₽» → раскрывает экран загрузки чека.
- Экран загрузки: drag-n-drop / клик по зоне (JPG/PNG/PDF, до 10 МБ), кнопка «Отправить чек на проверку», ссылка в поддержку.
- После отправки — переход на `/order-status` с сообщением «Чек на проверке».

### 5. Edge функции
- `create-sbp-order` — валидирует корзину (как `pay-with-balance`, без auto), создаёт `orders` со статусом `pending`/`payment_status='awaiting'`, списывает баланс если есть (как часть оплаты), создаёт `sbp_payments` со статусом `awaiting_receipt`. Возвращает orderId и сумму в ₽.
- `submit-sbp-receipt` — принимает orderId + файл чека (base64), грузит в bucket, обновляет `sbp_payments.status='pending_review'`, шлёт уведомление админам в Telegram с кнопкой «Открыть заявку».
- `sbp-admin-action` (внутренняя, вызывается из бота) — при approve: переводит заказ в paid, резервирует инвентарь, отправляет товар покупателю; при reject: возвращает баланс, шлёт сообщение покупателю.

### 6. Telegram бот — раздел «Заявки»
- В `admin/menu.ts` добавить кнопку «📥 Заявки СБП» (callback `a:sbp`).
- Новый файл `admin/sbp.ts`: список pending_review → карточка с фото чека, инфой заказа, суммой ₽ / $, реквизитами, кнопки «✅ Принять» / «❌ Отклонить (причина)».
- Принятие: вызывает существующую логику доставки (как `pay-with-balance` после оплаты), меняет статусы.
- Отклонение: причина → пользователю в ЛС, статус rejected, возврат баланса.

### 7. Telegram бот — Настройки «Реквизиты СБП»
В `admin/settings.ts`:
- Новая кнопка «🏦 Реквизиты СБП» в меню настроек → отдельный экран с текущими значениями и кнопками редактирования: Банк / Карта / ФИО / Телефон. FSM по аналогии с `se:edit:*`.

### 8. Ограничения
- СБП недоступен для Stars / Telegram Premium (auto-товары) — только CryptoBot/баланс.
- СБП недоступен в быстрой покупке (direct checkout) — только из корзины.

### Технические детали
- Курс константа `USD_RUB_RATE = 80` в shared utils edge-функций и `src/lib/sbp.ts`.
- Чек грузится через `supabase.storage.from('sbp-receipts').upload()` с path `{telegram_id}/{order_id}.{ext}`.
- Уведомление админу содержит ссылку на чек через signed URL (TTL 7 дней).
- При approve используется тот же путь доставки, что и CryptoBot webhook (вынести/переиспользовать логику резервирования инвентаря).
